# MySQL逻辑备份mysqldump

# mysqldump 概述

mysqldump 是 MySQL 自带的逻辑备份工具。可以保证数据的一致性和服务的可用性。 它的备份原理是通过协议连接到 MySQL 数据库，将需要备份的数据查询出来，将查询出的数据转 换成对应的 insert 语句，当我们需要还原这些数据时，只要执行这些 insert 语句，即可将对应的 数据还原。

# 备份命令

  • 1、命令格式
mysqldump [选项] 数据库名 [表名] > 脚本名
1
mysqldump [选项] --数据库名 [选项 表名] > 脚本名
1
mysqldump [选项] --all-databases [选项] > 脚本名
1

  • 2、选项说明

  • 3、备份实例

备份所有数据库：

[root@i-xrjnk70d ~]# mysqldump -uroot -p --all-databases > /backup/mysqldump/all.db
1

备份指定数据库：

[root@i-xrjnk70d ~]# mysqldump -uroot -p test > /backup/mysqldump/test.db
1

备份指定数据库指定表(多个表以空格间隔)

[root@i-xrjnk70d ~]# mysqldump -uroot -p mysql db event > /backup/mysqldump/2table.db
1

备份指定数据库排除某些表

[root@i-xrjnk70d ~]# mysqldump -uroot -p test --ignore-table=test.t1 --ignoretable=test.t2 > /backup/mysqldump/test2.db
1
  • 4、还原命令

    • a) 系统行命令

      [root@i-xrjnk70d ~]# mysqladmin -uroot -p create db_name 
[root@i-xrjnk70d ~]# mysql -uroot -p db_name < /backup/mysqldump/db_name.db
      1
      2

      在导入备份数据库前，db_name如果没有，是需要创建的；而且与db_name.db中数据库名是一 样的才可以导入。

    • b) soure 方法

      mysql > use db_name
mysql > source /backup/mysqldump/db_name.db
      1
      2

6)跳过 DELETE 和 DROP 语句

下面的操作就要小心翼翼了，不能一下子把二进制日志里面全备以后的操作全部恢复，一旦全部恢 复了，那开发删除操作也会恢复，我们只能跳过误操作的地方。

[root@i-xrjnk70d ~]# sed -n '22p' /backup/mysql/2019-11-2812-mysql-all.sql 
CHANGE MASTER TO MASTER_LOG_FILE='bin_log.000002', MASTER_LOG_POS=154;
